# 核心实现说明

## FTPCommandParser

`FTPCommandParser` 将客户端控制连接中的一行文本解析成 `FTPCommand`：

- 命令动词统一转为枚举。
- 参数保留空格，适配包含空格的目录名。
- 空命令会抛出 `emptyCommand`。

## FTPPathResolver

`FTPPathResolver` 是路径安全边界：

- FTP 绝对路径从共享根目录解析。
- FTP 相对路径基于当前 FTP 目录解析。
- 遇到试图越过根目录的 `..` 会抛出 `escapesRoot`。

## FTPServer / FTPSession

`FTPServer` 负责监听控制端口。每个客户端连接会创建一个 `FTPSession`。

`FTPSession` 负责：

- 发送 `220` 问候。
- 解析和处理控制命令。
- 维护当前目录、认证状态、主动/被动数据连接。
- 对 `LIST`、`NLST`、`MLSD`、`RETR`、`STOR` 等命令打开数据连接。
- 通过共享传输限流器控制上传/下载并发数，默认上限为 8。

## 日志设计

`LogStore` 是 actor：

- 内存中保留最近日志，供 UI 实时显示。
- 同步写入日志文件。
- 密码命令在进入日志前脱敏。
