# Repository Instructions

## User Release Preference

When a user-facing app change is completed and verified, also complete the release/update path unless the user explicitly says not to:

- Update the installed app in `/Applications`.
- Update GitHub with the final changes.
- Update GitHub Releases using `gh`.

Before doing release steps, verify the app build and tests relevant to the change. If a release step is blocked, report the exact blocker and the command/output that proves it.
